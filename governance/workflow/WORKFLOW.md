# Hermes 项目标准工作流

> 状态：⏳ Phase 1 讨论中 | 讨论参与：小艾/PM/QA/Coco/Wiki
>
> 适用范围：所有 hermes-agent 项目（代码、文档、skill、配置）

## 七阶段流水线

```
📋计划 ──→ 💬讨论 ──→ 📐设计 ──→ 💻编码 ──→ 🔍审查 ──→ ✅验收 ──→ 📝复盘
 小艾       全员        PM        Coco       QA        PM        全员
```

## 各阶段详解

### 📋 计划（Plan）
- **负责人**：小艾（架构师）
- **输入**：用户需求
- **产出**：GitHub Issue + .task 文件
- **门槛**：小艾签发，Issue 含验收标准

### 💬 讨论（Discuss）
- **负责人**：全员（PM/QA/Coco/Wiki）
- **输入**：计划阶段的 Issue
- **活动**：各自从角色角度审方案、提意见、反复迭代
- **产出**：统一意见（Issue Comment 记录）
- **门槛**：三人全部同意，有分歧继续讨论

### 📐 设计（Design）
- **负责人**：PM-agent
- **输入**：讨论结论
- **产出**：DESIGN.md / 技术方案 / 模板
- **门槛**：QA 审查通过

### 💻 编码（Code）
- **负责人**：Coco-agent
- **输入**：设计文档 + 验收标准
- **产出**：PR + 测试通过
- **门槛**：CI 通过，无编译/测试错误

### 🔍 审查（Review）
- **负责人**：QA-agent
- **输入**：PR + 设计文档
- **活动**：逐项对照验收标准检查
- **产出**：审查报告（通过/不通过 + 修改建议）
- **门槛**：全部达标，不通过回退编码（最多 3 轮）

### ✅ 验收（Accept）
- **负责人**：PM-agent
- **输入**：审查报告 + PR
- **活动**：汇总验证，更新 .task 状态
- **产出**：status=done, quality=passed
- **门槛**：小艾确认

### 📝 复盘（Retrospect）
- **负责人**：全员
- **输入**：项目全链路记录
- **产出**：复盘文档（顺畅点/卡壳点/改进项）
- **门槛**：无（每次必做）

## 阶段状态绑定

| 阶段 | Issue Label | .task status |
|------|------------|-------------|
| 计划 | `phase:plan` | pending→assigned |
| 讨论 | `phase:discuss` | assigned |
| 设计 | `phase:design` | running |
| 编码 | `phase:code` | running |
| 审查 | `phase:review` | needs_review |
| 验收 | `phase:accept` | done |
| 复盘 | `phase:retro` | done |

## 核心规则

1. **不到门槛不进下一阶段** — 无例外
2. **审查不过回退编码** — 最多 3 轮，超了升小艾
3. **架构变更/新 agent/数据方向** — PM 必须暂停汇报小艾
4. **复盘每次必做** — 卡点实时记录到 /tmp/hermes-retro-notes.md
5. **Wiki 知识更新** — Issue 关闭触发 wiki 同步
